​1⁰ Se você desenvolveu algo e não testou, não faça o commit na branch main. Em vez disso, crie uma branch secundária e realize o commit nela. Isso evita que outras pessoas deem continuidade ao projeto ou façam apresentações com erros.
2⁰ ​Realize os commits via terminal ou ferramentas apropriadas, sem "copiar e arrastar" arquivos. O método manual frequentemente causa o esquecimento de arquivos importantes ou a duplicação desnecessária de pastas.
​3⁰ Se você sabe que o código está ruim ou se não tentou resolver um problema, avise no grupo.
​4⁰ Leia o código antes de subir. É evidente para a equipe quando alguém não revisou o que escreveu
